## Contexte du site actuel

LaReponse est le nouveau nom de l’entreprise (ex-Hex Makina) et marque une fusion assumée entre deux pôles : hexmakina.be (le professionnel, orienté exécution, systèmes, livraison) et liebrex.net (le philosophe, orienté sens, langage, principes). Le site doit donc tenir une ligne de crête : une proposition de valeur très concrète (réduire coûts, dépendances, charge et empreinte) portée par une pensée structurée (principes, discipline, méthode), sans tomber ni dans le jargon technique, ni dans l’abstraction.

Le positionnement central reste : quitter les frameworks et solutions standard pour produire une réponse adaptée à chaque projet, plus économique, plus écologique, et sans payer le prix des dépendances (complexité, maintenance, fragilité, coûts cachés). Ce qui change avec la fusion, c’est que la “réponse” n’est pas seulement technique : elle est aussi conceptuelle (comment on pense le problème, comment on nomme, comment on décide, comment on structure).

## Conséquence directe sur l’UX

Le site doit gérer deux modes de lecture, sans créer deux sites :

Un mode “business” rapide : comprendre en 20–40 secondes ce que vous faites, pour qui, et comment vous contacter.
Un mode “principes” plus profond : comprendre la philosophie (ADDBAD et le reste), la discipline de design, et pourquoi cela produit de meilleurs systèmes (durables, sobres, adaptés).

La navigation et la hiérarchie doivent donc permettre de “survoler” puis “plonger” au bon endroit, sans friction.

## Enjeux éditoriaux

Le risque principal est la confusion (ou la dilution) si les deux voix sont mélangées sans architecture claire. La solution est une séparation nette mais perméable :

Voix professionnelle : promesse, offres, méthode appliquée, livrables, contact.
Voix philosophique : principes, vocabulaire, posture, justification (le “pourquoi”), exemples de pensée.

Le site doit relier les deux par des ponts courts (encarts, citations, définitions, micro-exemples), plutôt que par de longs textes hybrides.

## Structure de contenu attendue

Même sans URLs exactes, la structure la plus solide est :

Entrée “LaReponse” : synthèse (promesse + différence) + CTA contact.
Section “Offre” : ce que vous livrez (automatisation, documents, systèmes, plateformes).
Section “Méthode” : ADDBAD (étapes, discipline, garanties).
Section “Principes” : extraits structurés de liebrex (courts, indexés, navigables).
Section “Preuves” : mini cas / outputs / exemples (même très légers).
Contact + mentions.

One-page avec ancres ou multi-pages (Offre / Méthode / Principes / Cas / Contact) : les deux sont compatibles, mais l’important est le découplage “lecture rapide” vs “lecture profonde”.

## Projets LaReponse (liste)

À ce stade, la liste fiable côté “projets” est la liste de réalisations/types de systèmes proposés (pas un portfolio client) :

Automatisation calibrée.
Crawlers, parsers, traitement de documents.
Systèmes intégrés de planification, reporting, décision.
Portails sur mesure, plateformes business, e-commerce.

Si vous ajoutez un portfolio, il doit être traité comme une extension contrôlée (quelques cas très courts) pour éviter de parasiter la clarté.

## Enjeux DEV

Le site doit incarner la promesse “moins de dépendances” : léger, rapide, durable, maintenable. Le contenu doit être facilement éditable (fichiers, données structurées simples), et l’accessibilité doit être irréprochable (c’est un site de lecture, donc c’est un signal de sérieux).
